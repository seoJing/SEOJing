<Subtitle level={2}>버그 1: 코드 전체보기가 세로로 나온다</Subtitle>

<Paragraph>
  모바일 프레젠테이션 모드에서 코드블록의 "전체보기"를 누르면
  <strong>FullscreenView</strong>가 세로 방향으로 표시되는 문제가 있었다.
  프레젠테이션은 <strong>rotate(90deg)</strong>로 화면 전체를 가로로 돌려놓은
  상태인데, FullscreenView는 그 정보를 모른 채 body 기준
  <strong>fixed inset-0</strong>으로 붙어버리니 방향이 안 맞는 것이다.
</Paragraph>

<Paragraph>
  해결은 FullscreenView에 <strong>zIndex</strong>와 <strong>rotate</strong>{" "}
  prop을 추가하는 방향으로 갔다. 프레젠테이션 컴포넌트에서 FullscreenView를 열
  때
  <strong>
    rotate={"{"}needsRotation{"}"}
  </strong>
  과
  <strong>
    zIndex={"{"}60{"}"}
  </strong>
  을 넘겨주면, FullscreenView 내부에서 회전 여부와 z-index를 외부 컨텍스트에
  맞게 처리한다.
</Paragraph>

<Subtitle level={2}>버그 2: 프레젠테이션 종료 후 스크롤이 멈춘다</Subtitle>

<Paragraph>
  이건 재현 경로가 꽤 길었다. 프레젠테이션 진입 -> 코드 전체보기 열기 -> 코드
  닫기 -> 프레젠테이션 종료. 이 순서대로 하면 페이지 스크롤이 완전히 먹통이
  된다.
</Paragraph>

<Paragraph>
  원인을 추적해 보니 <strong>useEffect</strong> 의존성 배열에
  <strong>fullscreenCode</strong> 상태가 들어가 있었다. 코드 전체보기를 열고
  닫을 때마다 effect의 cleanup과 재실행이 반복되면서 <strong>overflow</strong>{" "}
  복원 순서가 꼬여버린 것이다.
</Paragraph>

<Paragraph>
  구체적으로 말하면, cleanup에서 overflow를 원래 값으로 돌리는 로직이 있는데
  코드 전체보기가 닫힐 때 cleanup이 한 번, 재실행 시 다시 hidden으로 설정되는
  사이클이 돈다. 프레젠테이션이 아직 열려 있는 상태에서 overflow가 잠깐 풀렸다가
  다시 잠기는데, 이때 저장해 둔 "원래 값"이 hidden으로 덮어씌워진다. 결국
  프레젠테이션 종료 시 hidden을 복원해 버리니 스크롤이 안 되는 것이다.
</Paragraph>

<Paragraph>
  해결은 간단했다. <strong>fullscreenCode</strong>를 <strong>ref</strong>로
  바꿔서 최신값을 참조하되, 의존성 배열에서는 제거했다. effect가 불필요하게
  재실행되지 않으니 overflow 복원 순서도 정상으로 돌아왔다.
</Paragraph>

<Subtitle level={2}>정리</Subtitle>

<Paragraph>
  테스트 커버리지 올리기와 버그 잡기를 병행한 하루였다. 특히 두 번째 버그는
  useEffect 의존성 배열이 만들어내는 부수효과의 좋은 사례라서 기억해 둘 만하다.
  상태값을 의존성에 넣으면 effect 사이클이 예상보다 자주 돌 수 있고, 그 안에서
  DOM을 직접 조작하는 코드가 있으면 순서 문제가 생기기 쉽다.
</Paragraph>
