- <strong>`repeat(auto-fill, minmax(280px, 1fr))`</strong>: 미디어 쿼리 없이

<Subtitle level={3}>JS 첫 시작 핵심 복습</Subtitle>

<Paragraph>
  이번 주 범위인 JS 타입, 변수, 호이스팅에서 자주 헷갈리는 포인트를 코드와 함께
  정리합니다.
</Paragraph>

<Subtitle level={3}>undefined vs null — 누가 준 값인가?</Subtitle>

<Paragraph>
  <strong>undefined</strong>는 JS 엔진이 "아직 값을 안 넣었네"라고 자동으로
  넣어주는 것입니다. <strong>null</strong>은 개발자가 "여기는 일부러
  비워놨어"라고 직접 넣는 것입니다. 비유하면 undefined는 택배가 아직 안 온 상태,
  null은 "빈 상자를 보낸" 상태입니다.
</Paragraph>

```js
let a; // 선언만 하고 값을 안 넣음
console.log(a); // undefined — JS 엔진이 넣어줌

let b = null; // 개발자가 의도적으로 "비어있음"을 표시
console.log(b); // null
```

<Subtitle level={3}>var의 함수 스코프 — 블록을 무시하는 변수</Subtitle>

<Paragraph>
  `var`는 블록(`if`, `for`)을 스코프 경계로 인식하지 않습니다. 이게 실질적으로
  어떤 문제를 만드는지 코드로 보겠습니다.
</Paragraph>

```js
// var — if 블록을 무시하고 밖에서도 살아있음
if (true) {
  var secret = "비밀번호123";
}
console.log(secret); // "비밀번호123" — if 밖에서 접근 가능

// let — 블록 안에서만 존재
if (true) {
  let safe = "안전한값";
}
console.log(safe); // ReferenceError — 블록 밖에서 접근 불가
```

<Subtitle level={3}>호이스팅 시각화 — 엔진이 실제로 처리하는 순서</Subtitle>

<Paragraph>
  호이스팅은 "선언이 위로 끌어올려진다"라고 배우지만, 실제로 코드가 이동하는
  것은 아닙니다. JS 엔진이 실행 전에 선언을 미리 읽어두는 것입니다.
</Paragraph>

```js
// 우리가 작성한 코드
console.log(x); // undefined — 에러 아님!
var x = 5;
console.log(x); // 5

// JS 엔진이 이해하는 순서 (개념적)
// var x;              ← 선언만 끌어올림, 값은 undefined
// console.log(x);     ← undefined
// x = 5;              ← 여기서 값 할당
// console.log(x);     ← 5
```

```js
// let/const는 다르게 동작
console.log(y); // ReferenceError! — TDZ(일시적 사각지대)
let y = 5;
// 선언은 인식하지만, 초기화 전까지 접근을 막음
```

<Subtitle level={3}>NaN === NaN이 false인 이유</Subtitle>

<Paragraph>
  <strong>NaN(Not a Number)</strong>은 "계산 실패"를 뜻합니다. 실패한 계산 두
  개가 같은 실패인지 비교하는 건 의미가 없으므로, JS 표준에서 NaN은 자기
  자신과도 같지 않다고 정의했습니다. NaN 여부 확인에는 반드시 `Number.isNaN()`을
  사용합니다.
</Paragraph>

```js
NaN === NaN; // false — 유일하게 자기 자신과 같지 않은 값
Number.isNaN(NaN); // true — 올바른 확인 방법
Number.isNaN("hello"); // false — 문자열은 NaN이 아님
isNaN("hello"); // true — 전역 isNaN은 타입 변환을 하므로 위험!
```

<Subtitle level={2}>2. 저번 주 과제 정답 및 풀이</Subtitle>

<Subtitle level={3}>학습 자료 5주차 과제</Subtitle>

<Paragraph>
  <strong>Q1. `null`과 `undefined`의 차이를 설명해주세요.</strong>
</Paragraph>

<Paragraph>
  둘 다 "값이 없음"을 나타내지만 주체가 다릅니다. <strong>undefined</strong>는
  JS 엔진이 자동으로 할당하는 값으로, 변수를 선언만 하고 값을 넣지 않았을 때
  자동으로 들어갑니다. <strong>null</strong>은 개발자가 의도적으로 "이 변수는
  비어있다"고 명시할 때 사용합니다. 실무에서는 변수를 의도적으로 비울 때 null을
  쓰고, undefined는 직접 할당하지 않는 것이 권장됩니다.
</Paragraph>

<Paragraph>
  <strong>Q2. `==`와 `===`의 차이는 무엇인가요?</strong>
</Paragraph>

<Paragraph>
  <strong>`==`</strong>는 비교 전에 타입을 자동으로 변환(암묵적 강제 변환)한 후
  비교합니다. 그래서 `0 == false`가 `true`가 되는 등 직관적이지 않은 결과가
  나옵니다. <strong>`===`</strong>는 타입 변환 없이 값과 타입이 모두 같아야
  `true`를 반환합니다. 예상치 못한 버그를 방지하기 위해 실무에서는 항상 `===`를
  사용합니다.
</Paragraph>

<Paragraph>
  이번 주 범위와 관련된 실제 면접 질문들입니다. 정답을 외우기보다는 자신만의

</Paragraph>

<Subtitle level={3}>
  Q1. `null`, `undefined`, undeclared(선언되지 않은 변수)의 차이는 무엇인가요?
</Subtitle>

- <strong>`undefined`</strong>: 변수가 선언됐지만 값이 할당되지 않은 상태. JS
  엔진이 자동 부여.
- <strong>`null`</strong>: 개발자가 의도적으로 "값 없음"을 명시한 상태. `typeof
  null`은 버그로 인해 `'object'`를 반환.
- <strong>undeclared</strong>: 선언 자체가 없는 변수. `typeof`로 접근하면
  `'undefined'`를 반환하지만, 직접 참조하면 `ReferenceError` 발생.

```js
let a;
console.log(a); // undefined
console.log(typeof a); // 'undefined'

let b = null;
console.log(b); // null
console.log(typeof b); // 'object' ← 버그

console.log(typeof c); // 'undefined' (선언 안 했지만 typeof는 안전)
console.log(c); // ReferenceError: c is not defined
```

<Subtitle level={3}>Q2. `var`, `let`, `const`의 차이를 설명해주세요.</Subtitle>

<Paragraph>
  세 가지 모두 변수 선언 키워드지만 스코프, 재선언, 재할당, 호이스팅 동작이
  다릅니다.
</Paragraph>

- <strong>`var`</strong>: 함수 스코프, 재선언 가능, 호이스팅 시 `undefined`로
  초기화. 예측하기 어려워 현재는 사용하지 않습니다.
- <strong>`let`</strong>: 블록 스코프, 재선언 불가, 재할당 가능. TDZ로 인해 선언
  전 접근 시 `ReferenceError`.
- <strong>`const`</strong>: 블록 스코프, 재선언·재할당 모두 불가. 단,
  객체·배열의 내부 값은 변경 가능.

<Paragraph>
  면접 팁: "기본적으로 const를 쓰고, 재할당이 필요한 경우에만 let을 쓴다. var는
  쓰지 않는다"고 정리하면 좋습니다.
</Paragraph>

<Subtitle level={3}>Q3. `==`와 `===`의 차이는 무엇인가요?</Subtitle>

<Paragraph>
  <strong>`==`(동등 연산자)</strong>는 비교 전에 두 값의 타입을 맞추는 암묵적
  타입 변환을 수행합니다. 그래서 `0 == false`, `'' == false`, `null ==
  undefined`가 모두 `true`가 됩니다.
  <strong>`===`(일치 연산자)</strong>는 타입 변환 없이 값과 타입이 모두 같아야
  `true`를 반환합니다. 예측 불가능한 버그를 방지하기 위해 실무에서는 항상
  `===`를 사용합니다.
</Paragraph>
