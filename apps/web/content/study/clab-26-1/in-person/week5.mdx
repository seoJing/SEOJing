<Subtitle level={2}>1. 저번 주 복습</Subtitle>

<Paragraph>
  지난주에는 CSS 레이아웃의 핵심인 <strong>Position</strong>과{" "}
  <strong>Flexbox</strong>를 배웠고, 학습 자료에서는{" "}
  <strong>Reflow/Repaint</strong>, <strong>미디어 쿼리</strong>,{" "}
  <strong>반응형 단위</strong>까지 다뤘습니다. 핵심만 빠르게 짚어보겠습니다.
</Paragraph>

<Subtitle level={3}>Position</Subtitle>

- <strong>`static`</strong>(기본값): 문서 흐름대로, top/left 등 무시
- <strong>`relative`</strong>: 원래 위치 기준으로 이동, 원래 자리는 유지
- <strong>`absolute`</strong>: positioned 조상 기준, 문서 흐름에서 빠짐
- <strong>`fixed`</strong>: 뷰포트 기준, 스크롤해도 고정

<Subtitle level={3}>Flexbox</Subtitle>

- <strong>`justify-content`</strong>: 주축(Main Axis) 정렬
- <strong>`align-items`</strong>: 교차축(Cross Axis) 정렬
- <strong>`flex: 1`</strong>: 남은 공간 전부 차지
- <strong>`flex-wrap: wrap`</strong>: 넘치면 줄바꿈

<Subtitle level={3}>학습 자료에서 다룬 추가 개념</Subtitle>

- <strong>Reflow/Repaint</strong>: 브라우저 렌더링 3단계(Layout → Paint →
  Composite). 애니메이션에는 `transform`과 `opacity`가 성능에 유리합니다.
- <strong>미디어 쿼리(@media)</strong>: 화면 크기에 따라 스타일을 분기합니다.
  모바일 퍼스트(`min-width`) 전략이 업계 표준입니다.
- <strong>반응형 단위</strong>: `vw`, `vh`, `dvh`. 모바일 전체 화면에는
  `100dvh`를 사용하면 주소창 문제를 해결할 수 있습니다.

<Subtitle level={2}>2. 저번 주 과제 정답 및 풀이</Subtitle>

<Subtitle level={3}>학습 자료 4주차 과제</Subtitle>

<Paragraph>
  <strong>
    Q1. `#C`에 `top: 0; left: 0;`을 주면, `#C`는 `#A`, `#B`, 뷰포트 중 어느 것을
    기준으로 배치될까요? 그 이유는 무엇인가요?
  </strong>
</Paragraph>

<Paragraph>
  <strong>`#A`를 기준</strong>으로 배치됩니다. `#C`는 `position: absolute`이므로
  가장 가까운 positioned 조상을 찾습니다. `#B`는 `position: static`이라
  건너뛰고, `#A`가 `position: relative`이므로 `#A`가 기준이 됩니다.
</Paragraph>

<Paragraph>
  <strong>
    Q2. `#E`에 `top: 10px; left: 20px;`이 적용되어 있습니다. `#E`는 `#C`, `#D`
    중 어느 것을 기준으로 배치될까요? 그 이유는 무엇인가요?
  </strong>
</Paragraph>

<Paragraph>
  <strong>`#D`를 기준</strong>으로 배치됩니다. `#E`는 `position: absolute`이고,
  바로 위 부모인 `#D`가 `position: relative`이므로 `#D`가 기준이 됩니다. `#C`는
  `position: absolute`이긴 하지만, `#D`가 더 가까운 positioned 조상입니다.
</Paragraph>

<Paragraph>
  <strong>
    Q3. 만약 `#D`의 `position: relative`를 `position: static`으로 바꾸면, `#E`의
    기준은 어떻게 달라질까요?
  </strong>
</Paragraph>

<Paragraph>
  `#D`가 `static`이 되면 건너뛰게 되고, 다음으로 가까운 positioned 조상인{" "}
  <strong>`#C`(`position: absolute`)가 기준</strong>이 됩니다. `absolute`도
  `static`이 아니므로 positioned 조상에 해당합니다.
</Paragraph>

<Subtitle level={2}>3. 면접 질문</Subtitle>

<Paragraph>
  이번 주는 CSS 마무리 주차인 만큼, 4주차 범위뿐만 아니라 HTML/CSS 전체를
  아우르는 면접 질문을 다뤄보겠습니다.
</Paragraph>

<Subtitle level={3}>Q1. CSS Grid와 Flexbox의 차이를 설명해주세요.</Subtitle>

<Paragraph>
  둘 다 CSS 레이아웃 시스템이지만, <strong>다루는 차원</strong>이 다릅니다.
</Paragraph>

- <strong>Flexbox</strong>: <strong>1차원</strong> 레이아웃입니다. 가로 또는
  세로, 한 방향으로 요소를 배치합니다. 내비게이션 바, 카드 한 줄 배치 등에
  적합합니다.
- <strong>CSS Grid</strong>: <strong>2차원</strong> 레이아웃입니다. 행과 열을
  동시에 제어할 수 있습니다. 대시보드, 사이드바+메인 레이아웃 등에 적합합니다.

<Paragraph>
  면접 팁: "한 방향이면 Flex, 두 방향이면 Grid"라고 요약한 뒤, 실무에서는 페이지
  전체를 Grid로, 내부 컴포넌트는 Flexbox로 섞어 쓴다고 답하면 좋습니다.
</Paragraph>

<Subtitle level={3}>
  Q2. 반응형 웹 디자인에서 모바일 퍼스트(Mobile First) 접근법이란 무엇인가요?
</Subtitle>

<Paragraph>
  모바일(작은 화면)을 <strong>기본 스타일로 작성</strong>하고, `min-width`
  미디어 쿼리로 점점 큰 화면에 대응하는 방식입니다.
</Paragraph>

- <strong>이유</strong>: 전 세계 웹 트래픽의 약 60%가 모바일이고, 단순한 것에서
  복잡한 것으로 확장하는 것이 반대 방향보다 쉽습니다.
- <strong>반대</strong>: 데스크톱 퍼스트는 `max-width`로 줄여나가는 방식입니다.
  레거시 프로젝트에서 볼 수 있지만 현재는 권장되지 않습니다.

<Subtitle level={3}>
  Q3. `<script>`, `<script defer>`, `<script async>`의 차이를 설명해주세요.
</Subtitle>

<Paragraph>
  셋 다 JavaScript를 로드하지만, <strong>HTML 파싱과의 관계</strong>가 다릅니다.
</Paragraph>

- <strong>`<script>`</strong>: HTML 파싱을 멈추고 스크립트를 다운로드·실행합니다.
- <strong>`<script defer>`</strong>: 백그라운드에서 다운로드하면서 HTML 파싱을
계속합니다. HTML 파싱 완료 후 <strong>작성 순서대로</strong> 실행합니다.
- <strong>`<script async>`</strong>: 백그라운드에서 다운로드하되, 다운로드 완료
즉시 실행합니다. 실행 순서가 <strong>보장되지 않습니다</strong>.

<Paragraph>
  면접 팁: "대부분의 경우 defer가 최적이고, 독립적인 스크립트(분석 도구 등)에만
  async를 쓴다"고 정리하면 간결합니다.
</Paragraph>

<Subtitle level={2}>4. 보충: CSS Grid — 2차원 레이아웃</Subtitle>

<Paragraph>
  Flexbox는 한 방향(가로 또는 세로)으로 요소를 배치하는{" "}
  <strong>1차원 레이아웃</strong>입니다. 하지만 가로와 세로를 동시에 제어해야
  하는 상황이 있습니다. 예를 들어 대시보드처럼 행과 열이 동시에 중요한
  레이아웃을 만들 때는 <strong>CSS Grid</strong>가 적합합니다.
</Paragraph>

<Subtitle level={3}>Grid 기초: 행과 열을 한 번에 정의</Subtitle>

```css
/* 3열 × 자동 행 그리드 */
.dashboard {
  display: grid;
  grid-template-columns: 1fr 1fr 1fr; /* 3개의 동일 너비 열 */
  gap: 16px; /* 행·열 간격 */
}

/* repeat()으로 간결하게 */
.dashboard {
  display: grid;
  grid-template-columns: repeat(3, 1fr);
  gap: 16px;
}
```

<Paragraph>
  `1fr`은 <strong>fraction(비율)</strong>의 약자로, 남은 공간을 비율로 나눕니다.
  `1fr 1fr 1fr`은 3등분, `1fr 2fr`은 1:2 비율입니다. Flexbox의 `flex: 1`과
  비슷한 개념이지만 열 단위로 적용됩니다.
</Paragraph>

<Subtitle level={3}>Grid vs Flexbox — 언제 뭘 쓸까?</Subtitle>

| 상황                           | 추천                        | 이유                     |
| :----------------------------- | :-------------------------- | :----------------------- |
| 내비게이션 바 (가로 한 줄)     | <strong>Flexbox</strong>    | 한 방향 정렬             |
| 카드 목록 (가로 나열 + 줄바꿈) | <strong>둘 다 가능</strong> | Flex + wrap 또는 Grid    |
| 대시보드 (행·열 동시 제어)     | <strong>Grid</strong>       | 2차원 배치               |
| 사이드바 + 메인 콘텐츠         | <strong>Grid</strong>       | 열 너비 고정 + 유동 조합 |
| 요소 하나를 정중앙 배치        | <strong>둘 다 가능</strong> | 취향 차이                |

```css
/* 실무에서 자주 보는 Grid 패턴: 사이드바 + 메인 */
.layout {
  display: grid;
  grid-template-columns: 250px 1fr; /* 사이드바 250px 고정, 나머지 유동 */
  min-height: 100vh;
}
```

<Paragraph>
  핵심 기준은 간단합니다. <strong>"한 방향이면 Flex, 두 방향이면 Grid"</strong>.
  실무에서는 둘을 섞어서 쓰는 경우가 대부분입니다. 전체 페이지 레이아웃은
  Grid로, 내부 컴포넌트는 Flexbox로 구성하는 패턴이 일반적입니다. Tailwind
  CSS에서는 `grid grid-cols-3 gap-4`처럼 유틸리티 클래스로 Grid를 선언하므로
  직접 CSS를 작성할 일은 줄어들지만, `1fr`이나 `auto-fill` 같은 개념을 이해해야
  Tailwind의 `grid-cols-[repeat(auto-fill,minmax(280px,1fr))]` 같은 커스텀 값도
  자유롭게 쓸 수 있습니다.
</Paragraph>

<Subtitle level={3}>Grid + 미디어 쿼리 = 반응형 레이아웃</Subtitle>

<Paragraph>
  4주차 학습 자료에서 미디어 쿼리를 배웠습니다. Grid와 미디어 쿼리를 조합하면
  화면 크기에 따라 열 수가 자동으로 바뀌는 반응형 레이아웃을 쉽게 만들 수
  있습니다.
</Paragraph>

```css
/* 모바일: 1열, 태블릿: 2열, 데스크톱: 3열 */
.card-grid {
  display: grid;
  grid-template-columns: 1fr;
  gap: 16px;
}

@media (min-width: 768px) {
  .card-grid {
    grid-template-columns: repeat(2, 1fr);
  }
}

@media (min-width: 1024px) {
  .card-grid {
    grid-template-columns: repeat(3, 1fr);
  }
}
```

<Paragraph>
  또는 미디어 쿼리 없이도 <strong>`auto-fill`과 `minmax()`</strong>를 사용하면
  브라우저가 알아서 열 수를 조절합니다.
</Paragraph>

```css
/* 자동 반응형: 최소 280px, 남는 공간은 균등 분배 */
.card-grid {
  display: grid;
  grid-template-columns: repeat(auto-fill, minmax(280px, 1fr));
  gap: 16px;
}
```

<Paragraph>
  이 한 줄이면 화면 너비에 따라 카드가 자동으로 1열, 2열, 3열로 재배치됩니다.
  미디어 쿼리를 직접 쓰는 것보다 코드가 훨씬 간결하면서도 유연합니다.
</Paragraph>

<Subtitle level={3}>반응형 단위: vw, vh, dvh</Subtitle>

<Paragraph>
  반응형 레이아웃에서 자주 쓰이는 뷰포트 단위도 알아두면 좋습니다.
</Paragraph>

- <strong>`vw` (viewport width)</strong>: 뷰포트 너비의 1%. `100vw`는 화면 전체
  너비입니다.
- <strong>`vh` (viewport height)</strong>: 뷰포트 높이의 1%. `100vh`는 화면 전체
  높이입니다.
- <strong>`dvh` (dynamic viewport height)</strong>: 모바일 브라우저에서 주소창이
  숨겨지거나 나타날 때 높이가 동적으로 변합니다. `100vh`는 주소창을 무시하고
  고정 높이를 잡지만, `100dvh`는 실제 보이는 영역에 맞춰 변합니다.

<Paragraph>
  모바일에서 `height: 100vh`를 쓰면 주소창에 가려져 콘텐츠가 잘리는 문제가 자주
  발생합니다. 이때 `100dvh`를 사용하면 해결됩니다. 최신 브라우저에서 모두
  지원하므로, 모바일 전체 화면 레이아웃에는 `dvh`를 사용하는 것이 좋습니다.
</Paragraph>
