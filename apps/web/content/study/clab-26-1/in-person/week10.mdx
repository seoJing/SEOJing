<Subtitle level={3}>클로저(Closure) — "함수가 기억하는 배낭"</Subtitle>

<Paragraph>
  클로저를 비유하면, 함수가 태어날 때 주변 변수를{" "}
  <strong>배낭에 담아가는 것</strong>
  입니다. 함수가 어디서 실행되든, 태어났을 때 담아간 배낭(렉시컬 환경)은 그대로
  들고 다닙니다.
</Paragraph>

```js
function makeGreeting(greeting) {
  // greeting이 배낭에 담김
  return function (name) {
    // 이 함수가 어디서 실행되든 greeting을 기억
    return `${greeting}, ${name}!`;
  };
}

const hello = makeGreeting("안녕하세요"); // greeting = "안녕하세요" 담김
const bye = makeGreeting("잘 가요"); // greeting = "잘 가요" 담김

hello("철수"); // "안녕하세요, 철수!"
bye("영희"); // "잘 가요, 영희!"
```

<Paragraph>
  <strong>React에서의 클로저</strong>: `useState`의 상태값, 이벤트 핸들러 안의
  변수가 모두 클로저로 동작합니다. 나중에 React를 배우면 이 개념이 바로
  연결됩니다.
</Paragraph>

```js
// React에서 클로저가 쓰이는 패턴 (미리보기)
function Counter() {
  const [count, setCount] = useState(0);

  const handleClick = () => {
    // handleClick은 count를 클로저로 기억
    setCount(count + 1);
  };

  return <button onClick={handleClick}>{count}</button>;
}
```

<Subtitle level={3}>비동기 처리 — "주문하고 번호표 받기"</Subtitle>

<Paragraph>
  비동기를 비유하면 <strong>카페에서 주문하는 것</strong>과 같습니다.
  주문(요청)을 넣고 번호표(Promise)를 받습니다. 음료가 나올 때까지 다른 일을
  하다가, 번호가 불리면(fulfilled) 음료를 받습니다. 만약 재료가 없으면(rejected)
  "죄송합니다"라는 답을 받습니다.
</Paragraph>

```js
// Promise = 번호표
const order = fetch("/api/coffee"); // 주문 접수 → 번호표(Promise) 반환

// async/await = 번호표를 들고 기다리기
async function getCoffee() {
  try {
    const response = await fetch("/api/coffee"); // 기다림
    const data = await response.json(); // 음료 수령
    return data;
  } catch (err) {
    console.error("주문 실패:", err); // 재료 없음
  }
}
```

<Subtitle level={3}>이벤트 루프 — 실행 순서 퀴즈</Subtitle>

<Paragraph>
  아래 코드의 실행 순서를 맞춰보세요.{" "}
  <strong>콜 스택 → 마이크로태스크(Promise) → 매크로태스크(setTimeout)</strong>{" "}
  순서를 기억하면 됩니다.
</Paragraph>

```js
console.log("1"); // 콜 스택 — 즉시 실행

setTimeout(() => console.log("2"), 0); // 매크로태스크 큐

Promise.resolve().then(() => console.log("3")); // 마이크로태스크 큐

console.log("4"); // 콜 스택 — 즉시 실행

// 출력 순서: 1, 4, 3, 2
```

<Paragraph>
  클로저는{" "}
  <strong>
    함수가 선언된 렉시컬 환경을 기억하여, 외부 함수가 종료된 후에도 외부 변수에
    접근할 수 있는 특성
  </strong>
  입니다. 활용 사례로는 데이터 은닉이 있습니다. `makeCounter()` 함수 내부의
  `count` 변수는 외부에서 직접 접근할 수 없지만, 반환된 클로저 함수를 통해서만
  조작할 수 있습니다.
</Paragraph>

<Paragraph>
  <strong>
    Q2. async/await와 Promise의 관계를 설명해주세요. async 함수의 반환값은 어떤
    타입인가요?
  </strong>
</Paragraph>

<Paragraph>
  `async/await`은 Promise를 기반으로 한{" "}
  <strong>문법적 설탕(syntactic sugar)</strong>입니다. 내부적으로는 Promise를
  사용하며, 비동기 코드를 동기 코드처럼 읽기 쉽게 작성할 수 있게 해줍니다.
  `async` 함수는 항상 <strong>Promise를 반환</strong>합니다. `return 42`를 해도
  `Promise.resolve(42)`와 동일하게 동작합니다.
</Paragraph>

<Subtitle level={3}>
  Q1. map과 filter의 차이는 무엇이고, 언제 사용하나요?
</Subtitle>

<Paragraph>
  둘 다 원본 배열을 변경하지 않고 <strong>새 배열을 반환</strong>하는 순수
  함수입니다.
</Paragraph>

- <strong>`map`</strong>: 각 요소를 <strong>변환</strong>하여 같은 길이의 새
  배열을 반환. "모든 요소에 같은 작업 적용"
- <strong>`filter`</strong>: 조건에 맞는 요소만 <strong>걸러내서</strong> 더
  짧거나 같은 길이의 새 배열을 반환. "조건에 맞는 것만 선택"

```js
const numbers = [1, 2, 3, 4, 5];

// map — 변환: 모든 요소를 2배로
numbers.map((n) => n * 2); // [2, 4, 6, 8, 10]

// filter — 필터링: 짝수만 선택
numbers.filter((n) => n % 2 === 0); // [2, 4]

// 체이닝: filter 먼저 → map
// 짝수만 골라서(2개) 2배 하기 vs 전체(5개) 2배 후 짝수 고르기
numbers.filter((n) => n % 2 === 0).map((n) => n * 2); // [4, 8]
```

<Paragraph>
  면접 팁: "React에서 `map`은 리스트 렌더링에, `filter`는 조건부 필터링에 매일
  사용합니다. 체이닝할 때 `filter`를 먼저 하면 처리할 요소 수가 줄어들어 더
  효율적입니다"라고 연결하면 좋습니다.
</Paragraph>

<Subtitle level={3}>Q2. 구조 분해 할당이 React에서 어떻게 쓰이나요?</Subtitle>

<Paragraph>
  구조 분해 할당은{" "}
  <strong>배열이나 객체의 값을 개별 변수로 한 번에 추출</strong>하는 문법입니다.
  React에서는 두 가지 핵심 패턴에 사용됩니다.
</Paragraph>

```jsx
// 1. props 수신 — 객체 구조 분해
function UserCard({ name, age, role = "user" }) {
  return (
    <div>
      {name} ({age}) - {role}
    </div>
  );
}

// 2. useState — 배열 구조 분해
const [count, setCount] = useState(0);
// useState가 [현재값, 변경함수] 배열을 반환하고,
// 구조 분해로 각각 변수에 할당

// 3. API 응답에서 필요한 값만 추출
const { data, error, isLoading } = useSWR("/api/user");
```

<Paragraph>
  면접 팁: "기본값 지정(`role = 'user'`), 이름 변경(`data: userData`), Rest
  파라미터(`{ name, ...others }`)까지 알고 있으면 코드가 훨씬 간결해집니다"라고
  답하면 좋습니다.
</Paragraph>

<Subtitle level={3}>
  Q3. 옵셔널 체이닝(?.)과 null 병합 연산자(??)를 설명해주세요. ||와 ??의 차이는?
</Subtitle>

<Paragraph>
  <strong>옵셔널 체이닝(`?.`)</strong>: 앞의 값이 `null`이나 `undefined`이면
  에러 없이 `undefined`를 반환합니다. 중첩 객체에 안전하게 접근할 때 사용합니다.
</Paragraph>

<Paragraph>
  <strong>null 병합 연산자(`??`)</strong>: 왼쪽이 `null` 또는 `undefined`일 때만
  오른쪽 값을 반환합니다.
</Paragraph>

```js
const user = null;

// 옵셔널 체이닝 — TypeError 방지
user?.address?.city; // undefined (에러 없음)

// ?. + ?? 조합 — 안전한 접근 + 기본값
user?.profile?.avatar ?? "/default.png"; // "/default.png"

// ?? vs || — 핵심 차이
const count = 0;
count || "기본값"; // "기본값" — 0은 falsy라서 오른쪽 반환
count ?? "기본값"; // 0 — 0은 null/undefined가 아니므로 왼쪽 유지

const name = "";
name || "익명"; // "익명" — 빈 문자열은 falsy
name ?? "익명"; // "" — 빈 문자열은 null/undefined가 아님
```

<Paragraph>
  면접 팁: "`||`는 0, 빈 문자열, false도 falsy로 처리합니다. 수량이나 금액처럼
  0이 유효한 값인 경우 `??`를 써야 의도치 않게 기본값으로 바뀌지 않습니다"라고
  설명하면 실무 감각이 있다는 인상을 줍니다.
</Paragraph>
