<Paragraph>
  지난주에는 React에서 매일 쓰는 ES6+ 문법을 집중적으로 배웠습니다. 핵심만

<Subtitle level={3}>배열 고차함수 — React의 리스트 렌더링 기반</Subtitle>

- <strong>`map`</strong>: 각 요소를 변환 → 새 배열 반환. React에서 리스트
  렌더링의 핵심
- <strong>`filter`</strong>: 조건 충족 요소만 → 더 짧은 새 배열. 상품 목록
  필터링 등
- <strong>`reduce`</strong>: 배열 → 하나의 값. 합계, 그룹핑, 장바구니 금액 계산
  등

```js
const products = [
  { name: "사과", price: 1000, inStock: true },
  { name: "바나나", price: 500, inStock: false },
  { name: "포도", price: 3000, inStock: true },
];

// 재고 있는 상품만 → 이름만 추출
products.filter((p) => p.inStock).map((p) => p.name); // ['사과', '포도']
```

<Subtitle level={3}>구조 분해 — props 수신의 핵심</Subtitle>

- <strong>객체 구조 분해</strong>: `const { name, age } = user` — React props 수신
시 매일 사용
- <strong>배열 구조 분해</strong>: `const [count, setCount] = useState(0)` —
  useState 반환값

```js
// React props 수신 패턴
function UserCard({ name, age, isAdmin = false }) {
  // 기본값 설정까지 한 번에
  return (
    <div>
      {name} ({age}세) {isAdmin && "👑"}
    </div>
  );
}
```

<Subtitle level={3}>스프레드 & 옵셔널 체이닝</Subtitle>

- <strong>스프레드</strong>: `{ ...prev, age: 26 }` — 불변 패턴. `useState`에서
객체 상태 업데이트 시 핵심
- <strong>`?.`</strong>: `user?.address?.city` — null/undefined 안전 접근. API
  응답 처리 시 필수
- <strong>`??`</strong>: `user.age ?? "나이 미입력"` — null/undefined만 폴백.
  `||`와 달리 0·''도 통과

<Paragraph>

```js
const user = { name: "철수", age: 25 };

const { name, age, role = "guest" } = user;
```

```js
const user = { name: "영희" };

// user.profile이 없으면 안전하게 undefined 반환, ?? 로 기본값 적용
const bio = user?.profile?.bio ?? "소개글이 없습니다";
// "소개글이 없습니다"
```

<Subtitle level={2}>3. React란 무엇인가?</Subtitle>

<Paragraph>
  React는 Facebook이 만든{" "}
  <strong>UI를 만들기 위한 JavaScript 라이브러리</strong>
  입니다. 지금까지 배운 JS를 그대로 쓰되, UI를 <strong>컴포넌트 단위</strong>로
  나눠서 만드는 방식을 따릅니다.
</Paragraph>

- <strong>선언적(Declarative)</strong>: "이렇게 보여라"라고 선언. 어떻게 DOM을
  수정할지는 React가 알아서 처리
- <strong>컴포넌트 기반</strong>: 레고처럼 작은 조각을 조합해서 복잡한 UI 구성
- <strong>단방향 데이터 흐름</strong>: 부모 → 자식 방향. 예측 가능하고 디버깅
  쉬움

<Subtitle level={2}>4. 컴포넌트 — UI의 레고 블록</Subtitle>

<Paragraph>
  컴포넌트는 <strong>재사용 가능한 UI 조각</strong>입니다. 함수형 컴포넌트는
  props를 입력받아 JSX(화면 구조)를 반환하는 JS 함수입니다.
</Paragraph>

```jsx
// 가장 단순한 컴포넌트
function Greeting() {
  return <h1>안녕하세요!</h1>;
}

// 컴포넌트를 조합해서 사용
function App() {
  return (
    <div>
      <Greeting />
      <Greeting />
    </div>
  );
}
```

<Paragraph>
  컴포넌트 이름이 <strong>소문자</strong>이면 React는 HTML 태그로 인식합니다.
  컴포넌트는 반드시 대문자로 시작해야 합니다.
</Paragraph>

<Subtitle level={2}>5. JSX — JS 안에 HTML처럼</Subtitle>

<Paragraph>
  JSX는 JS 안에서 HTML처럼 작성하는 문법입니다. 실제로는{" "}
  <strong>React.createElement() 호출</strong>로 변환됩니다. 바벨(Babel)이 이
  변환을 담당합니다.
</Paragraph>

```jsx
// JSX 문법
const element = <h1 className="title">안녕하세요</h1>;

// 바벨이 변환하는 실제 코드
const element = React.createElement("h1", { className: "title" }, "안녕하세요");
```

<Subtitle level={3}>JSX 주요 규칙</Subtitle>

```jsx
// 1. JS 표현식은 {} 안에 작성
const name = "철수";
<h1>안녕하세요, {name}님!</h1>;
<p>1 + 1 = {1 + 1}</p>;

// 2. class 대신 className (class는 JS 예약어)
<div className="container">내용</div>;

// 3. 반드시 하나의 루트 요소로 감싸야 함
// 잘못된 예
return (
  <h1>제목</h1>
  <p>내용</p>
);

// 올바른 예 — Fragment로 감싸기 (실제 DOM 요소 없이)
return (
  <>
    <h1>제목</h1>
    <p>내용</p>
  </>
);

// 4. 조건부 렌더링
{isLoggedIn && <Dashboard />}           // 조건이 true일 때만 렌더링
{isLoggedIn ? <Dashboard /> : <Login />} // 삼항 연산자
```

<Subtitle level={2}>6. props — 부모에서 자식으로 데이터 전달</Subtitle>

<Paragraph>
  props는 부모 컴포넌트에서 자식 컴포넌트로 데이터를 전달하는 방법입니다.{" "}
  <strong>읽기 전용</strong>입니다 — 자식은 props를 수정할 수 없습니다.
</Paragraph>

```jsx
// 부모 → 자식으로 데이터 전달
function App() {
  return <UserCard name="김철수" age={25} isAdmin={true} />;
}

// 자식 컴포넌트에서 props 수신 (구조 분해 권장)
function UserCard({ name, age, isAdmin }) {
  return (
    <div>
      <h2>
        {name} ({age}세)
      </h2>
      {isAdmin && <span>관리자</span>}
    </div>
  );
}
```

<Subtitle level={3}>props로 리스트 렌더링하기</Subtitle>

```jsx
const users = [
  { id: 1, name: "철수", role: "admin" },
  { id: 2, name: "영희", role: "user" },
];

function App() {
  return (
    <ul>
      {users.map((user) => (
        // key는 React가 리스트를 효율적으로 업데이트하는 데 필요
        <UserCard key={user.id} name={user.name} role={user.role} />
      ))}
    </ul>
  );
}
```

<Paragraph>
  리스트 렌더링 시 <strong>`key` prop</strong>이 필수입니다. key가 없으면 경고가
  발생하고 성능 문제가 생깁니다. key는 리스트 안에서 고유한 값이어야 하며, 보통
  데이터의 id를 사용합니다.
</Paragraph>

<Subtitle level={3}>인증 미션 과제</Subtitle>

<Paragraph>
  <strong>
    Q1. 아래 요구사항을 만족하는 `Counter` 컴포넌트를 만들어보세요. (1) + 버튼을
    누르면 숫자가 1 올라감 (2) - 버튼을 누르면 숫자가 1 내려감 (3) 초기값은 0
    (4) 함수형 업데이트(`prev ={">"} ...`)를 사용할 것
  </strong>
</Paragraph>

```jsx
function Counter() {
  const [count, setCount] = useState(0);
  return (
    <div>
      <p>{count}</p>
      <button onClick={() => setCount((prev) => prev + 1)}>+</button>
      <button onClick={() => setCount((prev) => prev - 1)}>-</button>
    </div>
  );
}
```

<Paragraph>
  `useState`로 `count` 상태를 관리하고, 버튼의 `onClick` 핸들러에서 `setCount`를
  호출합니다. <strong>함수형 업데이트(`prev => prev + 1`)</strong>를 사용하면
  항상 최신 상태를 기반으로 업데이트됩니다.
</Paragraph>

---

<Paragraph>
  <strong>
    Q2. `users` 배열을 props로 받아 이름 목록을 렌더링하는 `UserList` 컴포넌트를
    만들어보세요. `key` prop도 올바르게 사용해야 합니다.
  </strong>
</Paragraph>

```jsx
function UserList({ users }) {
  return (
    <ul>
      {users.map((user) => (
        <li key={user.id}>{user.name}</li>
      ))}
    </ul>
  );
}
```

<Paragraph>
  `map()`으로 배열을 순회하면서 각 요소를 `<li>`로 변환합니다.{" "}
  <strong>`key`에는 배열 인덱스가 아닌 고유한 `id`</strong>를 사용합니다. 이
  패턴은 React에서 가장 자주 사용하는 리스트 렌더링 방식입니다.
</Paragraph>

<Paragraph>
  <strong>
    Q3. `isLoggedIn` boolean prop을 받아서 `true`이면 `<Dashboard />`
    컴포넌트를, `false`이면 `<Login />` 컴포넌트를 렌더링하는 `AuthGate`
    컴포넌트를 작성해보세요.
  </strong>
</Paragraph>

```jsx
function Dashboard() {
  return <h1>대시보드입니다</h1>;
}

function Login() {
  return <h1>로그인해주세요</h1>;
}

function AuthGate({ isLoggedIn }) {
  return isLoggedIn ? <Dashboard /> : <Login />;
}
```

<Paragraph>
  삼항 연산자(`조건 ? A : B`)를 사용한 조건부 렌더링 패턴입니다. 실제
  프로젝트에서 인증 상태에 따라 다른 화면을 보여줄 때 이 패턴을 자주 사용합니다.
</Paragraph>

<Subtitle level={2}>7. 면접 질문</Subtitle>

<Subtitle level={3}>Q1. React에서 컴포넌트란 무엇인가요?</Subtitle>

<Paragraph>
  컴포넌트는 <strong>재사용 가능한 UI 조각</strong>입니다. 함수형 컴포넌트는
  props를 입력받아 JSX(화면 구조)를 반환하는 JS 함수입니다. 레고 블록처럼 작은
  컴포넌트를 조합해서 복잡한 UI를 만듭니다.
</Paragraph>

<Paragraph>
  면접 팁: "컴포넌트는 단일 책임 원칙을 따르는 것이 좋습니다. 한 컴포넌트가 너무
  많은 역할을 하면 재사용성과 테스트 가능성이 떨어집니다"라고 더하면 좋습니다.
</Paragraph>

<Subtitle level={3}>Q2. JSX란 무엇이고 왜 사용하나요?</Subtitle>

<Paragraph>
  JSX는 JS 안에서 HTML처럼 UI를 작성하는 문법 확장입니다. 내부적으로
  React.createElement() 호출로 변환됩니다. UI 구조를 직관적으로 표현할 수 있고,
  JS 표현식을 {}로 바로 삽입할 수 있어서 조건부 렌더링, 리스트 렌더링이
  편합니다.
</Paragraph>

<Subtitle level={3}>Q3. props와 state의 차이는 무엇인가요?</Subtitle>

<Paragraph>
  <strong>props</strong>는 부모 컴포넌트가 자식에게 전달하는 데이터입니다.
  자식은 읽을 수만 있고 수정할 수 없습니다. <strong>state</strong>는 컴포넌트
  자신이 관리하는 내부 데이터입니다. state가 변경되면 컴포넌트가 리렌더링됩니다.
  state는 다음 주에 자세히 다룹니다.
</Paragraph>
