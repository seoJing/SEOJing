게시글 작성처럼 본문이 있는 요청은 @RequestBody로 받습니다. 여기서 PostCreateRequest가 요청 DTO입니다. 프론트엔드는 이런 JSON을 보내야 합니다. 이제 API 계약을 읽는 핵심이 나옵니다. 요청 DTO의 필드 이름이 곧 프론트엔드가 보내야 하는 JSON 이름이다. 따라서 AI가 만든 DTO를 보면 바로 프론트엔드 요청과 비교해야 합니다. 이런 DTO를 AI가 만들었다면 프론트엔드가 기존에 보내던 title, content, categoryId와 맞지 않습니다. 문법은 맞아도 계약은 틀린 코드입니다. ---
