" " 바로 하단의 더 알면 좋을 것들을 확인해보세요. --- 1. JS의 타입 시스템 — 다른 언어와 뭐가 다른가 Java나 C++에서는 변수를 선언할 때 타입을 명시합니다. " " 정적 타입 언어 라고 부릅니다. 반면 JS는 " " 동적 타입 언어 로, 변수에 어떤 타입의 값이든 넣을 수 있고 런타임에 타입이 결정됩니다. JS의 원시 타입은 7가지입니다: " " number, string, boolean, undefined, null, symbol, bigint ( symbol 과 bigint 는 지금 주차에서는 몰라도 됩니다. 심화 주차에서 다룹니다). 이 외에 참조 타입으로 object (배열, 함수, 일반 객체 포함)가 있습니다. number 타입의 특이점 대부분의 언어는 정수와 실수 타입이 분리되어 있습니다(int, float, double 등). JS는 number 하나 로 모든 숫자를 표현합니다. 내부적으로 IEEE 754 64비트 부동소수점을 사용하는데, 쉽게 말해 " " 컴퓨터가 소수를 2진수로 저장할 때 딱 맞아 떨어지지 않는 경우 가 생긴다는 뜻입니다. 여기서 유명한 버그가 나옵니다. 이는 JS만의 문제가 아니라 부동소수점 표현 방식 자체의 한계입니다. 실무에서 금액 계산 등 정밀도가 중요한 경우에는 정수로 변환하거나 라이브러리를 사용해야 합니다. number 타입에는 세 가지 특별한 값도 있습니다. undefined vs null — 둘 다 "없음"인데 왜 두 개? 다른 언어에서는 보통 null 하나로 "값 없음"을 표현하는데, JS에는 undefined와 null이 둘 다 있습니다. undefined : JS 엔진이 자동으로 할당. "변수는 있는데 아직 값이 없다" null : 개발자가 의도적으로 할당. "이 변수는 비어있다고 명시한다" 실무에서 권장하는 방식은 " " 변수를 의도적으로 비울 때는 null을 사용 하고, undefined는 직접 할당하지 않는 것입니다. undefined를 직접 할당하면 "아직 초기화 안 됨"인지 "의도적으로 비운 것"인지 구분이 안 되기 때문입니다. undefined가 "아직 초기화되지 않은 상태"를 어떻게 만들어내는지는 섹션 2의 var/let/const 호이스팅 파트에서 다시 만납니다. …
